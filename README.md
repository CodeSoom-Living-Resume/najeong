### Duxcord

덕스코드는 슬랙, 줌, 디스코드의 주요 기능들을 클론코딩한 프로젝트입니다.

- 2021.10.25 ~ 2021.12.03
- [깃허브](https://github.com/boostcampwm-2021/web09-Duxcord)
- [프로젝트 소개 노션](https://www.notion.so/ecb9c3c8db3346cd97ad5d35fde624c0)
- 사용 기술
  1. 프론트엔드: React, Redux, SWR, styled-components
  2. 백엔드: Node.js, express, MySQL, TypeORM
  3. 기타: Jest, socket.io, WebRTC

**프로젝트 내용**

1. 그룹 및 채널 단위로 **실시간 채팅과 화상회의**를 지원하는 협업툴입니다.
2. 채팅 기능
   - 사용자들은 채팅 채널에서 실시간으로 대화를 할 수 있습니다. 채팅 메시지에 좋아요와 댓글을 추가할 수 있습니다.
   - `socket.io` 라이브러리를 사용하여 실시간 채팅을 구현하였으며 각각의 그룹/채널/스레드에 대한 새로운 정보를 필요한 사용자에게만 전달하기 위해 `Room` 기능을 활용했습니다.
   - `함수형 프로그래밍`을 이용하여 무한 스크롤 비동기 제어 로직을 작성했습니다.
3. 화상 기능
   - `WebRTC` API를 이용해 소규모 화상 회의 기능을 구현했습니다.
   - 라이브러리를 사용하지 않고 볼륨 측정, 장비상태 공유 기능을 구현했습니다.
4. 기타 기능
   - `pre-signed url`를 이용하여 client에서 곧바로 Object Storage로 이미지를 업로드할 수 있도록 로직을 작성했습니다.
   - Jest로 100여개의 프론트엔드 유틸 및 백엔드 로직 단위 `테스트 코드`를 작성하여 코드에 대한 안정성을 확보했습니다.
5. 협업
   - 매일 스크럼과 회고를 진행하여 팀 노션과 프로젝트 wiki에 기록했습니다. 또한 팀 내외부로 코드리뷰를 진행했습니다.
   - 팀 내에서 스터디를 진행했고 개발일지를 토대로 기술을 공유했습니다.

**기획 의도와 결과**

교육 기간동안 회의와 수업을 위해 슬랙과 줌을 함께 사용하여 불편함을 겪었습니다. 하나의 플랫폼에서 협업 과정을 모두 해결하고 싶다는 생각을 하였고 협업으로 대표적으로 알려진 디스코드와 슬랙, 줌의 장점을 합한 서비스를 개발했습니다. 실제로 서비스가 완성이 된 후 멘토님과 덕스코드에서 멘토링을 시행하고 회의도 진행했습니다.

**고민과 해결**

이미지 업로드 기능을 구현할 때 서버를 통하지 않고 클라이언트에서 곧바로 이미지를 업로드할 수 있도록 pre-signed url을 이용했습니다.

클라이언트에서 Object Storage로 이미지를 바로 업로드하면 서버의 트래픽을 줄일 수 있을거라 생각하여 클라이언트에서 S3 객체로 직접 이미지를 업로드하도록 기능을 구현했습니다. 그러나 이 경우 번들링을 하고 배포를 하게 되면 API 키가 노출 될 수 있다는 문제가 있었습니다.

문제를 해결하기 위해 팀원들이 모여 스크럼을 진행했습니다. 일정 기간동안 자격 증명을 생략할 수 있는 pre-signed url을 이용하여 문제를 해결했습니다. 클라이언트가 서버에 pre-signed url 생성을 요청하면 서버는 S3 객체의 putObject 메소드를 사용할 수 있는 pre-signed url을 생성하여 클라이언트에 전달하고 클라이언트는 해당 url로 이미지를 업로드합니다.

페어 프로그래밍을 진행하여 해당 기능을 구현했습니다. 기능을 구현하지 않은 팀원들에게도 진행 상황과 문제 해결 과정을 공유하기 위해 프로젝트 wiki에 글을 작성했습니다.

또한 기능을 구현하는 과정에서 cors 에러가 여러번 발생했습니다. AllowOrigin을 설정하고 요청/응답 헤더를 분석하여 오류를 파악했습니다. 오류의 원인을 파악하기 위해 stack-overflow의 글을 참고하였고 문제를 해결했습니다.
